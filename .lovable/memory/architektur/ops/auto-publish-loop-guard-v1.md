# Memory: architektur/ops/auto-publish-loop-guard-v1
Updated: 2026-03-16

Der `package_auto_publish` Loop Guard verhindert endlose Job-Spam-Loops bei deterministischen Gate-Fehlern (z.B. `integrity_passed=false` ohne aktiven Autofix-Lauf). Nach 3 Cancellations innerhalb von 2 Stunden für dasselbe Paket wird der Step `auto_publish` auf `status=blocked` gesetzt, alle verbleibenden pending Jobs gecancelt, und eine P0-Notification erzeugt. Die `pipeline-helpers.ts` skippt Steps im Status `blocked` automatisch, sodass kein Re-Enqueue stattfindet. Die View `v_ops_auto_publish_blockers` bietet Ops-Sichtbarkeit auf alle blockierten oder aktiven Auto-Publish-Steps mit Cancel-Historie. Ein manueller Unblock (Step auf `queued` + `blocked_reason` entfernen) ist jederzeit möglich, nachdem die Root Cause (z.B. tier1_failed Lektionen, fehlende QC-Promotion) behoben wurde. Der Guard unterscheidet zwischen transienten Fehlern (Timeout, Rate-Limit → weiter retrybar) und deterministischen Gate-Fehlern (integrity_passed=false → Loop Guard greift).
